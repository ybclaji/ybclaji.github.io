## 注意
### 浮动问题
1. 浮动因为宽度不够掉下去的问题，把li的父亲ul修改到足够宽，就不会因为宽度不够li到下面去了
2. 浮动的父盒子如果不给高度，则高度为0 
#### 为什么清除浮动： 
1 父级没有高度 2 自己浮动 3 影响下面的布局，需要清除浮动 撑开高度
4. 清除发动的方式：给父元素添加双伪元素   
### 外边距合并问题
子盒子为浮动的盒子，不会有外边距合并问题，如果都是标准流盒子会发生塌陷
### 行高问题
子盒子会继承父盒子的行高
## 定位
### 为什么需要定位
标准流和浮动无法解决的问题需要浮动
1. 需要自由移动的盒子 覆盖在其他盒子上方
2. 滚动窗口时候，盒子固定在屏幕某个位置
### 定位组成
定位=定位模式+边偏移，定位模式指定定位方式，边偏移决定位置
#### 定位模式
通过position设置，static,relative,absolute,fixed,static是默认定位方式，无定位的意思，就是标准流
#### 边偏移
定位的盒子移动位置用到4个属性：top bottom left right
#### 相对定位
1. 相对于自身原来的位置移动
2. 原来在标准流中的位置继续占有，不脱离标准流
3. 最典型的应用是给绝对定位当父亲
#### 绝对定位
1. 相对祖先元素移动
2. 如果没有父元素或者父元素没有定位，则以浏览器窗口为父亲
3. 如果祖先元素有定位（只要不是静态定位），则相对最近的有定位的祖先元素移动，就是爸爸有定位就找爸爸，爸爸没有就找爷爷
4. 绝对定位不在占有原来的位置（脱离标准流）
### 子绝父相定位使用场景
子级用绝对定位不会占有原来的位置，父元素加定位限制子级在父级内显示，如果父级加绝对定位下面的布局会占有父级的位置，所以为了占有位置只能用相对定位
#### 固定定位
1. 元素固定于浏览器可视区的位置，页面滚动时，元素位置不变
2. 以浏览器可视窗口为参照点移动元素
3. 固定定位的盒子小技巧：固定定位的盒子贴着版心，
4. 缩放页面不变：定位偏移用百分比，再使用margin
#### 粘性定位sticky
相对定位和固定定位的混合，开始是相对定位，滚动后为固定定位，兼容性很差postion:sticky
1. 以浏览器可视窗口为参照点移动元素
2. 占有原来的位置
3. 必须添加偏移
#### 定位叠放顺序
为了明确盒子上下，要确定叠放次序z-index,正数负数0，越大越在上
只有定位的盒子才有这个属性，标准流和浮动都没有
#### 定位的拓展
1. 绝对定位的盒子水平居中：定位的盒子margin:auto是不起作用的,left:50%;margin-left:负值自己宽度的一半
2. 特殊特性：行内元素使用定位后可以加宽度高度
3. 块元素加了定位后，如果不给宽度和高度，默认大小是内容的大小，没加定位是标准流的特点占一行
4. 标准流2个盒子，上面的盒子浮动后会压住下面的盒子，但是不会压住盒子的文字，图片，因为浮动最初的目的是为了环绕文字图片
5. 绝对定位的盒子会完全压住下面的盒子的内容
#### 元素的显示与隐藏
1. display显示隐藏
   display:none 不再占有原来的位置
   display:block 转换为块级元素并且显示，
2. visibility 显示隐藏
   visibility:visible/hidden 继续占有原来的位置
3. overflow 溢出显示隐藏
   overflow:visible/hidden/scroll：添加滚动条,auto:溢出的时候添加滚动条
   有定位的盒子，慎用overflow:hidden
### 高级技巧
#### 使用精灵图
为了有效减少服务器接受和发送请求的次数，提高页面的加载速度，原理是
将一些小北京图像整合到一张大图，请求一次即可。
1. 主要针对背景图片使用
2. 移动背景图片位置，background-position
3. 移动的距离是x,y 坐标
4. 一般情况是向上，左移动，值为负
5. 缺点：更换麻烦，图片失真，图片大
#### 使用字体图标
1. 使用字体图标替代精灵图，展示的是图标，本质是字体，可以自定义编辑
2. 本质是文字，比较小
3. 几乎支持多有浏览器
4. 样式结构简单，不能完全替代精灵图
5. 使用：1 下载 2 字体图标的引入 3 字体图标的追加
6. 字体图标站： 1 icomoon http://icomoon.io  2  阿里iconfont 字库
7. 字体图标的引入： 1 把fonts文件夹放到根目录 2 字体图标的引入 @font-face 3 在需要的地方使用图标 4 给标签声明字体
8. 字体图标的追加

#### 常见布局技巧
#### 行内元素，行内块元素，块元素
1. 行内元素
　　行内元素最常使用的就是span，其他的只在特定功能下使用，修饰字体<b>和<i>标签，还有<sub>和<sup>这两个标签可以直接做出平方的效果，而不需要类似移动属性的帮助，很实用。
　　行内元素特征：(1)设置宽高无效
　　　　　　　　　(2)对margin仅设置左右方向有效，上下无效；padding设置上下左右都有效，即会撑大空间
　　　　　　　　  （3）不会自动进行换行
2. 块状元素
　　块状元素代表性的就是div，其他如p、nav、aside、header、footer、section、article、ul-li、address等等，都可以用div来实现。不过为了可以方便程序员解读代码，一般都会使用特定的语义化标签，使得代码可读性强，且便于查错。
　　块状元素特征：(1)能够识别宽高
　　　　　　　　　(2)margin和padding的上下左右均对其有效
　　　　　　　　　(3)可以自动换行
　　　　　　　　　(4)多个块状元素标签写在一起，默认排列方式为从上至下
3. 3.行内块状元素
　　行内块状元素综合了行内元素和块状元素的特性，但是各有取舍。因此行内块状元素在日常的使用中，由于其特性，使用的次数也比较多。
　　行内块状元素特征：(1)不自动换行
　　　　　　　　　　　(2)能够识别宽高
　　　　　　　　　　　(3)默认排列方式为从左到右
